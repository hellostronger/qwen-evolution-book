## 2.2 模型架构设计

### 核心架构

Qwen-1 基于 **Transformer** 架构，继承业界最佳实践并做出优化：

```
┌─────────────────────────────────────────────────────┐
│                Qwen-1 架构组件                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌────────────────────────────────────────────┐    │
│  │  Embedding Layer                           │    │
│  │  - Vocabulary Size: ~150K                  │    │
│  │  - 优化中英双语tokenization                 │    │
│  └────────────────────────────────────────────┘    │
│                      ↓                             │
│  ┌────────────────────────────────────────────┐    │
│  │  Transformer Layers (N layers)             │    │
│  │  ┌──────────────────────────────────────┐  │    │
│  │  │ Multi-Head Attention (MHA)           │  │    │
│  │  │ + RoPE Position Encoding             │  │    │
│  │  └──────────────────────────────────────┘  │    │
│  │  ┌──────────────────────────────────────┐  │    │
│  │  │ SwiGLU Activation                    │  │    │
│  │  └──────────────────────────────────────┘  │    │
│  │  ┌──────────────────────────────────────┐  │    │
│  │  │ RMSNorm Normalization                │  │    │
│  │  └──────────────────────────────────────┘  │    │
│  └────────────────────────────────────────────┘    │
│                      ↓                             │
│  ┌────────────────────────────────────────────┐    │
│  │  Output Layer                              │    │
│  │  - Language Model Head                     │    │
│  └────────────────────────────────────────────┘    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 关键技术选择

| 技术 | 说明 | 优势 |
|-----|------|------|
| **RoPE** | 旋转位置编码 | 支持长序列，无需额外位置embedding |
| **SwiGLU** | Swish + GLU组合激活 | 训练稳定性好，非线性能力强 |
| **RMSNorm** | Root Mean Square LayerNorm | 比 LayerNorm 计算更高效 |
| **BPE Tokenizer** | Byte Pair Encoding | 基于tiktoken，优化中英文 |

### RoPE 实现原理

**旋转位置编码（Rotary Position Embedding）**：

```python
# RoPE 简化实现
def apply_rotary_pos_emb(q, k, pos):
    """
    对 query 和 key 应用旋转位置编码
    
    Args:
        q: query tensor [batch, heads, seq_len, head_dim]
        k: key tensor [batch, heads, seq_len, head_dim]
        pos: position indices [seq_len]
    """
    # 将 head_dim 分成两半，进行旋转
    cos, sin = get_cos_sin_cache(pos, head_dim)
    
    # 旋转公式：
    # q_rotated = q * cos + rotate_half(q) * sin
    # k_rotated = k * cos + rotate_half(k) * sin
    
    return apply_rotation(q, k, cos, sin)
```

**优势**：
- 相对位置编码，更符合语言特性
- 支持任意长度序列（外推性好）
- 无需额外学习位置参数

## 2.3 多尺寸发布策略

### 版本矩阵

| 模型 | 参数量 | 层数 | Hidden Size | 定位 |
|-----|-------|------|-------------|------|
| **Qwen-7B** | 7B | 32 | 4096 | 轻量级，易部署 |
| **Qwen-14B** | 14B | 40 | 5120 | 平衡型，性价比高 |
| **Qwen-72B** | 72B | 80 | 8192 | 高性能，旗舰级 |

### 尺寸选择逻辑

```
                    性能
                      ↑
                      │
         Qwen-72B ●───┤ 旗舰级
                      │
         Qwen-14B ●───┤ 平衡型
                      │
         Qwen-7B  ●───┤ 轻量级
                      │
      ─────────────────┼────────────→ 部署成本
         低           中            高
```

**策略考量**：
- **7B**：单卡可运行，适合个人开发者与边缘部署
- **14B**：性能与成本平衡，适合大多数应用场景
- **72B**：展示最强能力，适合研究与高端服务

## 2.4 首次能力评估

### Benchmark 结果（Qwen-1 初版）

> 注：具体数值需参考官方发布，以下为大致范围

### 初版优势与不足

**优势**：
- ✅ 中文理解能力强（C-Eval表现优异）
- ✅ 多尺寸覆盖不同需求
- ✅ 开源策略吸引社区关注

**不足**：
- ⚠️ 数学能力相对薄弱
- ⚠️ 编程能力有待提升
- ⚠️ 推理效率需要优化
